# Архитектура упрощенного JSON файла (Входные данные для препроцессора)

Этот JSON представляет собой «чистый контент» формы. В нем **полностью отсутствуют** системные ключи (`id`, `key`, `slug`, `owner` и т.д.), метаданные публикации, настройки уведомлений и пустые значения (`null`). Вся техническая обертка генерируется позже на стороне препроцессора.

Глобальная структура файла — это **строгий массив объектов** `[ { ... }, { ... } ]`. Каждый объект в массиве представляет собой отдельную форму (тест или раздел).

---

## 1. Корневой объект формы

Каждый объект формы внутри основного массива содержит только два ключа:

* `formName` *(String)* — Название формы (берется из названия раздела/темы). *Использование ключа `name` строго запрещено.*
* `questions` *(Array)* — Массив объектов, содержащих вопросы и элементы формы. *Использование ключа `items` строго запрещено.*

---

## 2. Массив `questions` (Вопросы и элементы)

Каждый объект внутри массива `questions` — это отдельный, независимый вопрос. Использование вложенных групп (тип `series`) запрещено.

### Базовые поля (общие для всех типов)
* `type` *(String)* — Тип поля (`"text"`, `"choices"`, `"staticText"`).
* `view` *(String)* — Визуальное представление поля (`"textinput"`, `"textarea"`, `"radio"`, `"select"`, `"header"`).
* `title` *(String)* — Текст вопроса или заголовка. **Важно:** Текст должен быть очищен от порядковых номеров (1., 2.) и маркеров форматирования (например, звездочек `**`).
* `description` *(String, опционально)* — Дополнительные инструкции или перечисление вариантов для текстового ввода. Может содержать переносы строки `\n`.
* `attachment` *(Object, опционально)* — Объект с изображением к вопросу. Содержит единственный вложенный ключ `externalUrl` со строковой ссылкой на `avatars.mds.yandex.net`.
* `limitMin` / `limitMax` *(Integer, опционально)* — Ограничения количества вводимых символов (используется в вопросах на последовательность).

---

## 3. Специфические поля в зависимости от типа вопроса

### Тип А: Вопросы с выбором вариантов (`type: "choices"`)
Дополнительные поля для одиночного выбора (радиокнопки или выпадающие списки).
* `sort` *(String)* — Параметр перемешивания вариантов. По умолчанию всегда `"shuffle"`. Для вопросов "Истина/Ложь" или "Да/Нет" указывается `"natural"`.
* `options` *(Array)* — Массив вариантов ответа.
  * `text` *(String)* — Текст варианта. Не должен содержать нумерации (А., Б., В.) и знаков препинания на конце (кроме точки). Если это сгенерированный комбинированный ответ (из множественного выбора), части разделяются символами `;\n`.

### Тип Б: Текстовый ввод (`type: "text"`)
Используется для коротких ответов (`view: "textinput"`) или развернутых эссе (`view: "textarea"`).
* Массив `options` **не используется**. Ответ вписывается пользователем.
* Первые два вопроса в любой форме всегда имеют этот тип ("ФИО" и "Класс").

### Тип В: Заголовок (`type: "staticText"`)
Используется для группировки или инструкций перед серией однотипных заданий (например, "Истина/Ложь").
* Используется `view: "header"`.
* Отсутствуют массивы `options` и `quiz`.

---

## 4. Логика проверки и начисления баллов: Массив `quiz`

Присутствует **только** в тех вопросах, где есть однозначный правильный ответ и за которые начисляются баллы (отсутствует в `textarea`, "ФИО", "Класс" и `staticText`).

Для упрощенного ИИ-JSON массив `quiz` всегда содержит **только один объект** — эталонный правильный ответ:

* `value` *(String)* — Точный текст правильного ответа. 
  * Для `choices` должен посимвольно совпадать со значением `text` правильного варианта в массиве `options`. 
  * Для `text` (ввод с клавиатуры) содержит эталонную строку (например, слитно написанные буквы `"вдбаегзж"` или слово `"колодец, шалаш"`).
* `scores` *(Integer)* — Количество баллов за правильный ответ (обычно `1`).
* `correct` *(Boolean)* — Флаг правильности (всегда `true`).

---

## Пример эталонного ИИ-JSON (одна форма с двумя вопросами)

```json
[
  {
    "formName": "Тест по дисциплине «Военная история»",
    "questions": [
      {
        "type": "text",
        "view": "textinput",
        "title": "ФИО"
      },
      {
        "type": "choices",
        "view": "radio",
        "title": "Определите направление по рисунку:",
        "attachment": {
          "externalUrl": "[https://avatars.mds.yandex.net/get-forms/6197807/](https://avatars.mds.yandex.net/get-forms/6197807/)..."
        },
        "sort": "shuffle",
        "options": [
          { "text": "Север." },
          { "text": "Юг." }
        ],
        "quiz": [
          {
            "value": "Север.",
            "scores": 1,
            "correct": true
          }
        ]
      }
    ]
  }
]
