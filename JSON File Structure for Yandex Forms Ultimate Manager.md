# Архитектура JSON-файла для Yandex Forms Migrator

Глобальная структура файла представляет собой **массив объектов** `[ { ... }, { ... } ]`. Каждый объект в массиве — это отдельная независимая форма.

Ниже разобран каждый узел одного объекта формы (от корня до вложенных элементов).

---

## 1. Корневой уровень (Root Level)

Описывает базовую идентификацию формы и содержит все основные конфигурационные блоки строго на одном уровне иерархии.

* `originalId` *(String)* — Уникальный идентификатор формы (hex-строка, генерируется системой, например, `"69e12ef349af473552d5a9a9"`).
* `formName` *(String)* — Внутреннее техническое название формы (например, `"Разведка"` или `"Тактическая подготовка"`).
* `texts` *(Object)* — Объект с текстами интерфейса.
* `surveyData` *(Object)* — Объект системных настроек конструктора.
* `surveyInfo` *(Object)* — Объект настроек рендеринга для пользователя.
* `submitConditions` *(Array)* — Массив условий отправки формы.
* `quizSettings` *(Object)* — Объект глобальных настроек тестирования.
* `questions` *(Array)* — Основной массив с вопросами формы.
* `hooks` *(Array)* — Массив настроек автоматизаций и уведомлений.

---

## 2. Объект `texts` (Пользовательский интерфейс)

Содержит локализацию и тексты системных кнопок и экранов, которые видит пользователь.

* `submit` *(String)* — Текст кнопки отправки (обычно `"Отправить"`).
* `back` *(String)* — Текст кнопки возврата на предыдущую страницу (`"Назад"`).
* `next` *(String)* — Текст кнопки перехода на следующую страницу (`"Далее"`).
* `title` *(String)* — Заголовок экрана успешной отправки (`"Спасибо!"`).
* `subtitle` *(String)* — Подзаголовок экрана успешной отправки (`"Ваше сообщение отправлено."`).
* `redirect` *(String)* — Сообщение при автоматическом редиректе (если включено).

---

## 3. Объекты метаданных: `surveyData` и `surveyInfo`

Система разделяет настройки на два похожих объекта. `surveyData` использует формат `camelCase` и отвечает за хранение состояния в конструкторе (админке), а `surveyInfo` использует формат `snake_case` и отвечает за параметры рендеринга формы для конечного пользователя (фронтенд).

### 3.1. `surveyData` (Настройки конструктора)
* `name` *(String)* — Публичное название формы.
* `isPublished` *(Boolean)* — Статус публикации (`true` / `false`).
* `language` *(String)* — Язык формы (`"ru"`).
* `collabId` *(String)* — Идентификатор организации/пространства (UUID).
* `redirect` *(Object)* — Настройки перенаправления после заполнения:
    * `enabled` / `auto` *(Boolean)* — включение редиректа.
    * `url` *(String)* — куда перенаправлять.
    * `delay` *(Integer)* — задержка в миллисекундах (напр. `5000`).
* `group` / `groupId` *(Object / Integer)* — Опциональная привязка к папке/группе пользователя (ФИО автора, ID организации).

### 3.2. `surveyInfo` (Настройки рендеринга)
Содержит настройки поведения формы для респондента (используются строгие snake_case ключи):
* `is_published`, `language`, `collab_id` — дублируют параметры из `surveyData`.
* `is_public` *(Boolean)* — Доступна ли форма публично по ссылке.
* `answers_count` *(Integer)* — Текущее количество собранных ответов (при миграции обычно `0`).
* `allow_multiple_answers` *(Boolean)* — Разрешена ли повторная отправка формы одним пользователем (`true`).
* `show_last_answer` *(Boolean)* — Показывать ли пользователю его предыдущий ответ.
* `footer`, `teaser`, `share` *(Boolean)* — Отображение подвала, тизера и кнопок "Поделиться" Яндекса.
* `fill_again` *(Boolean)* — Показывать ли кнопку "Заполнить еще раз" после отправки.
* `need_auth` *(Boolean)* — Требуется ли авторизация в Яндексе для заполнения.
* `is_new_frontend` *(Boolean)* — Использование новой версии интерфейса форм (`true`).

---

## 4. Настройки квиза (Тестирования)

Логика начисления баллов. Настройки дублируются в корне как `quizSettings` и иногда внутри `surveyInfo.quiz`.

### Объект `quizSettings` (корневой)
* `showResults` *(Boolean)* — Показывать ли респонденту итоговый балл после отправки.
* `showCorrect` *(Boolean)* — Показывать ли, где были допущены ошибки (правильные ответы).
* `countingMethod` *(String)* — Метод подсчета. В примерах используется `"range"` (диапазон баллов).
* `minimumScore` *(Integer)* — Минимально возможный балл (обычно `0`).
* `totalScore` *(Integer)* — Максимально возможный балл за весь тест (сумма всех правильных ответов).
* `numberOfQuestions` *(Integer)* — Общее количество оцениваемых вопросов.
* `numberOfSegments` *(Integer)* — Количество сегментов результата (в примерах `2`).
* `messages` *(Array)* — Настройки сообщений в зависимости от набранных баллов (в примерах оставлены пустыми).

---

## 5. Массив `questions` (Ядро формы)

Это массив объектов, где каждый объект — это конкретный вопрос или текстовый блок в форме.

### Базовые параметры (общие для всех вопросов)
* `id` *(Integer)* — Уникальный числовой ID поля (например, `119137623`).
* `key` *(String)* — Уникальный строковый ключ поля (используется препроцессором, например, `"id-string-119137623"` или `"answer_choices_900897649..."`).
* `type` *(String)* — Тип данных (например, `"text"`, `"choices"`, `"staticText"`).
* `view` *(String)* — Элемент интерфейса (например, `"textinput"`, `"radio"`, `"header"`).
* `title` *(String)* — Текст самого вопроса.
* `description` *(String)* — (Опционально) Подзаголовок или инструкция к вопросу. Ожидает строку, может содержать переносы `\n`.
* `required` *(Boolean)* — Обязательно ли поле для заполнения (`true`/`false`).
* `hidden` *(Boolean)* — Скрыто ли поле от пользователя (`false`).
* `externalValidator` *(Boolean)* — Системный флаг внешней проверки, всегда должен быть выставлен в `false`.

### Тип 1: Текстовый ввод (Открытый ответ / ФИО)
Используется для коротких ответов или полей ввода данных пользователя.
* `type`: `"text"`
* `view`: `"textinput"`
* Если это вопрос с автопроверкой, внутри добавляется блок `quiz`.

### Тип 2: Выбор одного варианта (Радиокнопки)
Используется для классических тестовых вопросов.
* `type`: `"choices"`
* `view`: `"radio"`
* `widget`: `"radio"`
* `sort` *(String)* — Сортировка вариантов ответа. `"shuffle"` (перемешивать случайно) или `"natural"` (в порядке создания, используется для Истина/Ложь).
* `options` *(Array)* — Массив вариантов ответа. Каждый вариант содержит:
    * `id` *(Integer)* — ID варианта ответа.
    * `key` *(String)* — Ключ варианта (обычно совпадает с ID или генерируется как Timestamp).
    * `text` *(String)* — Текст варианта ответа.
    * `hidden` *(Boolean)* — Скрыт ли вариант (`false`).

### Тип 3: Заголовок / Текст без ввода (Static Text)
Используется для разбиения формы, инструкций (например, перед блоком "Истина/Ложь").
* `type`: `"staticText"`
* `view`: `"header"`
* Здесь нет ни `options`, ни `quiz`, ни `required`. Только `id`, `key`, `title` и базовые флаги сброшены в `false`.

---

## 6. Логика проверки: Массив `quiz` (Внутри `questions`)

Этот массив присутствует только в тех вопросах, за которые начисляются баллы. Внутри него лежит массив объектов (по объекту на каждый возможный вариант ответа или на правильный текст).

### Для вопросов типа `choices` (Выбор из списка)
Массив `quiz` зеркально отражает массив `options`, но содержит разбалловку:
* `value` *(String)* — Текст ответа (Должен СТРОГО совпадать с полем `text` в `options`).
* `scores` *(Integer)* — Количество баллов за выбор этого варианта (`1` для правильного, `0` для неправильного).
* `correct` *(Boolean)* — Флаг правильности (`true` / `false`).

*Пример:*
```json
"quiz": [
  { "scores": 1, "correct": true, "value": "Правильный ответ" },
  { "scores": 0, "correct": false, "value": "Неправильный ответ" }
]
```

### Для вопросов типа `text` (Свободный ввод с автопроверкой)

Массив содержит только один объект — эталонный правильный ответ.

* `value` *(String)* — Точный текст правильного ответа (например, `"Радиоигра"`).
* `scores` *(Integer)* — Баллы за точное совпадение (`1`).
* `correct` *(Boolean)* — (`true`).

---

## 7. Массив `hooks` (Автоматизации / Уведомления)

Описывает действия, которые форма выполняет после успешной отправки данных (на стороне бэкенда Яндекса).

* `id` *(Integer)* — ID группы хуков.
* `name` *(String)* — Название (обычно `"Группа действий"`).
* `active` *(Boolean)* — Включена ли эта группа действий.
* `subscriptions` *(Array)* — Массив конкретных действий (например, отправка писем).

### Описание подписки (Email-уведомление):

* `type`: `"email"`
* `active`: *(Boolean)* Включена ли отправка.
* `language`: `"ru"`
* `email_to_address`: Адрес получателя. В примерах используются **переменные-подстановки** (например, `{69e12eda068ff096f10010ff}`).
* `subject`: Тема письма с подстановками (`"Ответ на форму {ID}"`).
* `body`: Тело письма с подстановками.
* `variables` *(Array)* — Расшифровка переменных, используемых в письме. Связывает UUID-подобный ID с системной переменной Яндекса:
* `type: "form.name"` — Название формы.
* `type: "form.author_email"` — Email создателя/владельца формы.
* `type: "form.url"` — Публичная ссылка на форму.
* `type: "form.answer_url"` — Ссылка на конкретный ответ респондента в админке.

## Важные системные особенности (на основе примеров):

1. **Разделители в ответах**: При комбинировании вариантов множественного выбора используется разделитель `;\n` (точка с запятой и перенос строки) прямо внутри строки `text` и `value`.
2. **Отсутствие ID при генерации**: В случае создания кастомных скриптов не рекомендуется генерировать поля `id` и `key`. Яндекс Формы (или скрипты-препроцессоры) способны автоматически назначать строковые (`id-string-119...`, `id-enum-119...`) или временные (timestamp) ключи, а также целочисленные ID, которые нужны серверу Яндекса для сохранения структуры.
3. **Отсутствие null**: Если поле не заполнено, ключа просто нет, либо используется пустая строка `""` или `false`. Единственное исключение в корневых `surveyData`, где системные настройки типа `languages: null` присутствуют технически.
